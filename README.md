# 설비관리 대시보드 (구조 공유용)

설계 문서: `../_웹서비스설계/설계.md` — 아키텍처·데이터모델·화면·단계별 계획은 전부 거기에 있음.
이 README는 실행법만.

완전 클라이언트 사이드 앱. 서버 없음, 파일은 브라우저를 떠나지 않음.

## 실행

```bash
npm install
npm run dev       # http://localhost:5173
npm run build     # dist/ 생성, GitHub Pages / Cloudflare Pages에 그대로 배포
```

## 진행 상황

- **Phase 1 (뼈대 + 샘플 데이터)** — 완료. 라우팅, 다크테마, 가공 샘플 데이터,
  대시보드/설비목록/설비상세 화면 동작 확인(로컬 빌드+브라우저 검증 완료).
- **Phase 2 (파싱 파이프라인)** — 완료. 랜딩 드롭존 실연동(폴더 드래그+`webkitdirectory`
  폴더선택 버튼 둘 다), xlsx/pdf/hwpx/pptx/docx 변환기(`lib/convert.ts`, hwp는 브라우저
  미지원 안내로 폴백), 분류 로직(`lib/classify.ts`, sync_documents.py 포팅 — 상세는 §7
  포팅표 정정 참고), 3열 검토 화면(`UploadReview.tsx`, 인라인 수정+전체 반영),
  Dexie(IndexedDB) 영속화(새로고침해도 유지 확인됨). 실제 xlsx 파일로 폴더 드롭→검토→
  수정→반영→대시보드/설비상세 반영→새로고침 유지까지 전 과정 브라우저 검증 완료.
  Worker로 감싸지 않고 메인 스레드에서 직접 돌림(README 하단 "알려진 제약" 참고).
- **Phase 3 (통계·그래프·내보내기)** — 완료. stats.ts(Phase 1)에 이어 brain.ts 포팅
  (연쇄고장/설치코호트/동시다발, 대시보드 카드로 노출), `/graph` 2D·3D 관계 그래프
  (react-force-graph, 옵시디언 그래프뷰 톤), `/history` 이력 브라우저(필터+고아 이력 탭),
  `/settings` JSON·옵시디언 볼트 zip 내보내기 + JSON 가져오기. 로컬 빌드+브라우저 검증 완료
  (내보낸 zip의 frontmatter·와이키링크까지 실제로 열어서 확인함).
- **Phase 4 (다듬기)** — 완료. code-splitting, 사이드바 모바일 대응, 빈/에러 상태 다듬기
  (설비 상세 "찾을 수 없음" 상태에 마스코트+정확한 메시지 추가), framer-motion 마이크로
  애니메이션(카드 hover 리프트, KPI 숫자 카운트업, 페이지 전환 페이드) 전부 적용. Suspense
  경계를 Layout 안쪽으로 옮겨서 라우트 전환 중에도 사이드바가 안 사라지게 고침.

각 Phase의 완료 기준·세부 작업은 설계 문서 §8 참고.

### 확장 모듈 — 설비 레이아웃 매핑 (`/mapping`, 설계 문서 §11)

- **Phase A (캔버스 핵심)** — 완료. 도면 업로드, 줌/팬, 우측 툴바 드래그→드롭 배치,
  배치된 설비 재드래그, %좌표 localStorage 영속화(새로고침 유지 확인됨), 클릭 팝오버
  (상태색+온도 오버레이는 데모용 가상수치). 로컬 빌드+브라우저 검증 완료.
- **Phase B (스마트 레이어)** — 완료. 연결선(Equipment.연결설비 기반 자동 드로잉 +
  클릭 시 흐름 애니메이션, 저장 안 하고 매번 파생), 구역 지오펜싱(다각형 그리기 +
  이름 + 평균 가동률/에러율 집계, localStorage 영속화), 레이어 토글 4종(설비명/구역선/
  수치/연결선) 전부 실제 동작. 로컬 빌드+브라우저 검증 완료(연결선 선택, 구역 생성·통계·
  삭제, 토글 on/off, 새로고침 후 구역 유지까지 확인).
- **Phase C (유지보수·분석)** — 완료. 보기 모드 4종(일반/유지보수/히트맵/타임라인) 전부
  실제 전환됨. 유지보수 모드: 다음점검일 임박(7일 이내)·경과 설비에 🔧 배지, 클릭하면
  대기→진행중→완료 순환(localStorage 영속화). 히트맵 모드: 최근1년 고장건수 기반 반투명
  방사형 그라디언트로 위험 집중 구역 근사(별도 라이브러리 없이). 히스토리 타임라인: 하단
  슬라이더로 과거 시점 이동, 그 시점 직전 수리 이력 유무로 "그 무렵 이상 있었음"을
  역산 재현(확정 상태 기록 아님, 화면에 명시). 로컬 빌드+브라우저 검증 완료.

이제 설계.md §11 "설비 레이아웃 매핑" 모듈은 Phase A/B/C 전부 완료.

- **아이콘 크기 조절 (2026-07-20 추가)**: 도면 위 설비 토큰을 클릭하면 뜨는 팝오버에
  "－/슬라이더/＋"로 50~250% 범위에서 크기를 줄였다 키웠다 할 수 있음. 도면(`도면ID`)별로
  따로 저장되므로(`Placement.scale`) 같은 설비도 도면마다 다른 크기로 강조 가능. localStorage
  영속화 확인(재선택·새로고침 후에도 유지), Konva Group의 scaleX/scaleY로 원·라벨·수치·
  작업오더 배지까지 한 번에 비례 확대/축소.

## 부서 무관 배포 원칙

"역"/"로터리"/"시장" 같은 특정 부서 지명은 코드 어디에도 없음 — 사이트는 사용자가
드롭한 폴더명에서 그때그때 뽑는다(설계 문서 §4.3). 다른 부서에 배포하기 전 이 원칙이
지켜지고 있는지 항상 확인할 것.

## 알려진 제약 (Phase 4)

- ~~**번들 크기**~~ **해결(2026-07-19)** — 모든 라우트를 `React.lazy`로, 업로드
  파이프라인(xlsx/pdfjs/mammoth)은 `Landing.tsx`에서 첫 파일이 들어올 때만 동적
  import 하도록 바꿈. 초기 번들이 3.84MB → 357KB(gzip 113KB)로 줄었고, three.js
  (1.47MB)·파싱 라이브러리(1.16MB)·pdf.worker(1.25MB)는 각 라우트를 실제로 열거나
  파일을 올릴 때만 받아옴. 전체 라우트 이동 Playwright로 재확인, 콘솔 에러 0건
- ~~**사이드바가 모바일에서 안 접힘**~~ **해결(2026-07-19)** — 390px 뷰포트로 실제
  확인해보니 고정폭 사이드바(w-56) 때문에 본문이 166px로 눌려서 거의 못 씀. `Layout.tsx`에
  `md:` 기준 햄버거+드로어 오버레이 추가(데스크톱은 기존 그대로, `md:static md:translate-x-0`
  로 회귀 없음 확인). 대시보드/설비목록/이력/설정처럼 텍스트+차트 위주 화면은 이걸로 충분히
  볼만해짐
- **레이아웃 매핑(`/mapping`)·관계 그래프는 모바일 대응 범위 밖** — 캔버스 편집(줌/팬/
  드래그 배치) 자체가 좁은 화면에 안 맞는 도구라, 3열(제어패널+캔버스+자산목록) 레이아웃을
  모바일에서 그대로 열면 캔버스가 옆 패널들에 밀려 거의 안 보임. 설계.md도 애초에
  "그래프 뷰는 2D 고정"이라고 모바일 제약을 예견해뒀음 — 데스크톱 전용 도구로 그대로 두고,
  모바일에서 굳이 써야 한다면 그때 별도로 손볼 것(탭 전환 등)
- **파싱이 메인 스레드에서 돎** — 설계.md는 Web Worker를 지정했지만, pdfjs 자체 워커가
  무거운 부분은 이미 오프로드하고 개별 문서 단위 파싱은 빠른 편이라 이번엔 생략함.
  폴더가 수백 개 파일 규모로 커지면 그때 커스텀 Worker로 감싸는 걸 권장
- **hwp(구버전) 미지원** — hwpx로 재저장 안내만 함(설계.md §9에서 이미 예견한 제약)
- **버그 헌팅 중 발견한 것**: 브라우저 포팅 과정에서 실제 테스트 데이터로 원본 파이썬
  스크립트(`sync_documents.py`)의 잠재 버그 2건을 같이 찾아 고침 — ① 타입 분류가 "TEST-500"
  같은 모델명의 "ES" 부분일치로 승강기로 오분류되던 것(짧은 영문 약어 단어경계 확인 추가),
  ② 이 브라우저 포팅본 자체에 있었던 날짜 파싱 UTC/로컬시간 혼용으로 하루씩 밀리던 버그.
  둘 다 원본 스크립트와 이 저장소에 각각 반영됨
- **코드리뷰 패스 (2026-07-20)**: fork 서브에이전트로 `src/` 43개 파일 전체를 훑어 버그 3건 +
  정리 4건 발견, 전부 수정 후 Playwright로 재검증. 실제 버그: ①
  `Mapping.tsx`에서 도면을 전환해도 구역 그리기 중이던 draftPoints/선택 상태가 안 비워져서
  A 도면에서 그리다 만 구역이 B 도면에 그대로 붙을 수 있었음(전환 핸들러에서 상태 리셋
  추가). ② `EquipmentToken.tsx`의 작업오더 배지 클릭이 부모 Group의 드래그를 못 막아서
  배지를 누르면 상태 순환 대신 설비가 끌려다니는 문제(Konva는 드래그를 mousedown/
  touchstart에서 시작하므로 그 단계에서 cancelBubble 추가 — 대기→진행중→완료→대기 순환과
  토큰 위치 불변 둘 다 브라우저에서 확인). ③ `Landing.tsx` 드롭존에 파일이 아닌 것(텍스트
  선택 등)을 드롭해서 인식된 파일이 0건이면 `dragleave` 없이 바로 `drop`이 발생해서 강조
  테두리가 새로고침 전까지 안 풀리던
  문제(0건 처리 분기에서 모드 리셋 추가). 정리: `ConnectionLine.tsx` 죽은 ref 제거,

  안내 배너 추가, `vaultExport.ts`의 이력 파일명이 날짜/설비/유형/제목까지 같으면 zip 안에서
  서로 덮어쓰던 것(id를 파일명 끝에 추가해 유일성 보장)
