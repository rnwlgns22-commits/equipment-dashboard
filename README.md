# 설비관리 대시보드 (구조 공유용)

설계 문서: `../_웹서비스설계/설계.md` — 아키텍처·데이터모델·화면·단계별 계획은 전부 거기에 있음.
이 README는 실행법만.

완전 클라이언트 사이드 앱. 서버 없음, 파일은 브라우저를 떠나지 않음.

## 실행

```bash
npm install
npm run dev       # http://localhost:5173
npm run build     # dist/ 생성, GitHub Pages / Cloudflare Pages에 그대로 배포
```

## 진행 상황

- **Phase 1 (뼈대 + 샘플 데이터)** — 완료. 라우팅, 다크테마, 가공 샘플 데이터,
  대시보드/설비목록/설비상세 화면 동작 확인(로컬 빌드+브라우저 검증 완료).
- **Phase 2 (파싱 파이프라인)** — 미착수. 드롭존 실제 연동, Worker 변환기, 분류 포팅,
  검토 화면, IndexedDB 저장.
- **Phase 3 (통계·그래프·내보내기)** — stats.ts는 Phase 1에서 이미 포팅됨(failure_stats.py).
  brain.ts(연쇄고장/코호트/동시다발), 2D/3D 그래프 뷰, 마크다운 내보내기는 미착수.
- **Phase 4 (다듬기)** — 미착수.

각 Phase의 완료 기준·세부 작업은 설계 문서 §8 참고.

### 확장 모듈 — 설비 레이아웃 매핑 (`/mapping`, 설계 문서 §11)

- **Phase A (캔버스 핵심)** — 완료. 도면 업로드, 줌/팬, 우측 툴바 드래그→드롭 배치,
  배치된 설비 재드래그, %좌표 localStorage 영속화(새로고침 유지 확인됨), 클릭 팝오버
  (상태색+온도 오버레이는 데모용 가상수치). 로컬 빌드+브라우저 검증 완료.
- **Phase B (스마트 레이어)** — 완료. 연결선(Equipment.연결설비 기반 자동 드로잉 +
  클릭 시 흐름 애니메이션, 저장 안 하고 매번 파생), 구역 지오펜싱(다각형 그리기 +
  이름 + 평균 가동률/에러율 집계, localStorage 영속화), 레이어 토글 4종(설비명/구역선/
  수치/연결선) 전부 실제 동작. 로컬 빌드+브라우저 검증 완료(연결선 선택, 구역 생성·통계·
  삭제, 토글 on/off, 새로고침 후 구역 유지까지 확인).
- **Phase C (유지보수·분석)** — 미착수. 작업오더 아이콘, 히트맵 모드, 히스토리 타임라인.

## 부서 무관 배포 원칙

"역"/"로터리"/"시장" 같은 특정 부서 지명은 코드 어디에도 없음 — 사이트는 사용자가
드롭한 폴더명에서 그때그때 뽑는다(설계 문서 §4.3). 다른 부서에 배포하기 전 이 원칙이
지켜지고 있는지 항상 확인할 것.
