# 설비관리 대시보드 (구조 공유용)

설계 문서: `../_웹서비스설계/설계.md` — 아키텍처·데이터모델·화면·단계별 계획은 전부 거기에 있음.
이 README는 실행법만.

완전 클라이언트 사이드 앱. 서버 없음, 파일은 브라우저를 떠나지 않음.

## 실행

```bash
npm install
npm run dev       # http://localhost:5173
npm run build     # dist/ 생성, GitHub Pages / Cloudflare Pages에 그대로 배포
```

## 진행 상황

- **Phase 1 (뼈대 + 샘플 데이터)** — 완료. 라우팅, 다크테마, 가공 샘플 데이터,
  대시보드/설비목록/설비상세 화면 동작 확인(로컬 빌드+브라우저 검증 완료).
- **Phase 2 (파싱 파이프라인)** — 미착수. 드롭존 실제 연동, Worker 변환기, 분류 포팅,
  검토 화면, IndexedDB 저장.
- **Phase 3 (통계·그래프·내보내기)** — 완료. stats.ts(Phase 1)에 이어 brain.ts 포팅
  (연쇄고장/설치코호트/동시다발, 대시보드 카드로 노출), `/graph` 2D·3D 관계 그래프
  (react-force-graph, 옵시디언 그래프뷰 톤), `/history` 이력 브라우저(필터+고아 이력 탭),
  `/settings` JSON·옵시디언 볼트 zip 내보내기 + JSON 가져오기. 로컬 빌드+브라우저 검증 완료
  (내보낸 zip의 frontmatter·와이키링크까지 실제로 열어서 확인함).
- **Phase 4 (다듬기)** — 미착수. 남은 후보: `/graph`(three.js) code-split으로 초기 번들
  축소, 모바일 반응형, 빈/에러 상태 다듬기, framer-motion 마이크로 애니메이션.

각 Phase의 완료 기준·세부 작업은 설계 문서 §8 참고.

### 확장 모듈 — 설비 레이아웃 매핑 (`/mapping`, 설계 문서 §11)

- **Phase A (캔버스 핵심)** — 완료. 도면 업로드, 줌/팬, 우측 툴바 드래그→드롭 배치,
  배치된 설비 재드래그, %좌표 localStorage 영속화(새로고침 유지 확인됨), 클릭 팝오버
  (상태색+온도 오버레이는 데모용 가상수치). 로컬 빌드+브라우저 검증 완료.
- **Phase B (스마트 레이어)** — 완료. 연결선(Equipment.연결설비 기반 자동 드로잉 +
  클릭 시 흐름 애니메이션, 저장 안 하고 매번 파생), 구역 지오펜싱(다각형 그리기 +
  이름 + 평균 가동률/에러율 집계, localStorage 영속화), 레이어 토글 4종(설비명/구역선/
  수치/연결선) 전부 실제 동작. 로컬 빌드+브라우저 검증 완료(연결선 선택, 구역 생성·통계·
  삭제, 토글 on/off, 새로고침 후 구역 유지까지 확인).
- **Phase C (유지보수·분석)** — 완료. 보기 모드 4종(일반/유지보수/히트맵/타임라인) 전부
  실제 전환됨. 유지보수 모드: 다음점검일 임박(7일 이내)·경과 설비에 🔧 배지, 클릭하면
  대기→진행중→완료 순환(localStorage 영속화). 히트맵 모드: 최근1년 고장건수 기반 반투명
  방사형 그라디언트로 위험 집중 구역 근사(별도 라이브러리 없이). 히스토리 타임라인: 하단
  슬라이더로 과거 시점 이동, 그 시점 직전 수리 이력 유무로 "그 무렵 이상 있었음"을
  역산 재현(확정 상태 기록 아님, 화면에 명시). 로컬 빌드+브라우저 검증 완료.

이제 설계.md §11 "설비 레이아웃 매핑" 모듈은 Phase A/B/C 전부 완료.

## 마스코트

공단 마스코트 "뉴비"를 브랜딩 요소로 사용 중(부서 전용이 아니라 공단 전체 마스코트라 §4.3
지명 금지 원칙과 무관 — 오히려 여러 부서가 공유하는 도구에 어울리는 조직 공통 자산).

- 원본: 볼트 밖 `../2. 3D_캐릭터_파일/` (턴어라운드·표정·동작 PNG 모음, 2500×2500 고화질)
- 프로젝트에 쓰는 건 `sharp`로 리사이즈해 용량을 줄인 버전만 — `src/assets/mascot/`
  (greeting.png=인사, thumbsup.png=엄지척, surprised.png=당황). 원본을 그대로 쓰지 않음
- 쓰인 곳: 랜딩 페이지 인사말, 전역 플로팅 Q&A 도움말 위젯(`components/MascotHelp.tsx`),
  레이아웃 매핑 화면의 "도면 없음" 빈 상태
- 추가로 쓰고 싶으면 원본 폴더에서 다른 포즈를 골라 같은 방식(sharp로 리사이즈 후
  `src/assets/mascot/`에 저장)으로 추가할 것 — 2500px 원본을 그대로 import하지 말 것
  (번들 크기 급증)

## 부서 무관 배포 원칙

"역"/"로터리"/"시장" 같은 특정 부서 지명은 코드 어디에도 없음 — 사이트는 사용자가
드롭한 폴더명에서 그때그때 뽑는다(설계 문서 §4.3). 다른 부서에 배포하기 전 이 원칙이
지켜지고 있는지 항상 확인할 것.
